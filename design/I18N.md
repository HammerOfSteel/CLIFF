# CLIFF Internationalization (i18n)

## Overview

CLIFF supports multiple languages through a simple context-based translation system. Currently supported languages:
- **Swedish (sv)** - Default
- **English (en)**

## How It Works

### 1. Language Context

All translations are managed through `LanguageContext` located in `/design/src/contexts/LanguageContext.tsx`.

The context provides:
- `language`: Current language code ('sv' or 'en')
- `setLanguage(lang)`: Function to change language
- `t(key)`: Translation function

### 2. Using Translations in Components

```tsx
import { useLanguage } from '@/contexts/LanguageContext';

function MyComponent() {
  const { t, language } = useLanguage();
  
  return (
    <div>
      <h1>{t('common.title')}</h1>
      <button>{t('common.save')}</button>
    </div>
  );
}
```

### 3. Adding New Translations

Edit `/design/src/contexts/LanguageContext.tsx` and add keys to both `sv` and `en` objects:

```tsx
const translations = {
  sv: {
    'myFeature.title': 'Min Titel',
    'myFeature.description': 'Min beskrivning',
  },
  en: {
    'myFeature.title': 'My Title',
    'myFeature.description': 'My description',
  },
};
```

### 4. Language Switching

Users can change language in **Settings** page:
- 🇸🇪 Svenska (Swedish)
- 🇬🇧 English

The selected language is saved to `localStorage` and persists across sessions.

## Best Practices

1. **Use translation keys for all user-facing text**
   ```tsx
   // Good
   <h1>{t('profile.title')}</h1>
   
   // Avoid
   <h1>Profil</h1>
   ```

2. **Keep keys organized by feature/section**
   - Use dot notation: `feature.subfeature.element`
   - Group related translations together

3. **For dynamic content or story content**
   - Story titles, content, and user-generated text should NOT be translated
   - Only UI elements should use the translation system

4. **Fallback behavior**
   - If a key is not found, the key itself is returned
   - Always add keys to both languages

## Extending Language Support

To add a new language:

1. Add language code to `Language` type:
   ```tsx
   type Language = 'sv' | 'en' | 'de'; // Adding German
   ```

2. Add translations object:
   ```tsx
   const translations = {
     sv: { ... },
     en: { ... },
     de: {
       'nav.discover': 'Entdecken',
       // ... all other keys
     },
   };
   ```

3. Add language option to Settings page.
